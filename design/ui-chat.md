# 聊天区与输入框设计

> 从 [ui.md](ui.md) 延伸：聊天区承载完整用户视图，输入框负责用户与 March 的交互入口。

---

## 消息流

- 用户消息右对齐，AI 消息左对齐
- AI 输出流式渲染
- 工具调用以折叠摘要内联展示，默认收起，点击展开原始 input/output
- 完整对话历史，用户可上下翻阅

## 输入框

### 布局

```
┌─────────────────────────────────────────────────────┐
│  帮我重构 [auth.rs ×] 的认证逻辑…                    │
│                                                     │
├─────────────────────────────────────────────────────┤
│  [+]  [⚡ claude-sonnet-4-6 ∨]              [↵发送] │
└─────────────────────────────────────────────────────┘
```

- 输入区自动扩展高度，最高约 6 行后出现内部滚动
- `@` 引用的文件以 chip 形式内联显示，chip 上有 `×` 可移除
- `Enter` 发送，`Shift+Enter` 换行

---

### `@` mention

用户输入 `@` 触发 mention 面板，浮于输入框上方：

```
┌──────────────────────────┐
│  > auth                  │  ← fuzzy 搜索
│  src/auth.rs             │  ← 文件
│  src/auth/               │  ← 目录
└──────────────────────────┘
```

两类目标，行为不同：

| 类型 | 行为 |
|------|------|
| `@文件` | 加入 `open_files`，出现在右栏监控区 |
| `@目录` | 注入目录树结构到本条消息，不加入 `open_files` |

**目录树注入规则**：

- 遵守 `.gitignore` 过滤
- 超过 2,000 行截断，在消息头部附注 `[目录树共 X 项，已截断至 2000 行]`
- 只显示路径结构，不展开文件内容

### 模型选择器

```
[⚡ claude-sonnet-4-6 ∨]
   ↓ 点击展开
   ┌──────────────────────────┐
   │ ✓ claude-sonnet-4-6      │  ← 当前选中
   │   claude-opus-4-6        │
   │   gemini-2.5-pro         │
   │   ──────────────────     │
   │   管理 Provider...       │
   └──────────────────────────┘
```

- 列表来自已配置的 provider（`config.toml`）
- 每个 session 独立记录选择，不影响其他 session
- 图标为 provider 标识，短名为模型名

---

### `+` 菜单

```
[+] 点击展开
   ┌──────────────────┐
   │  选择文件…       │  → 文件选择器，等同 @文件
   │  选择目录…       │  → 目录选择器，等同 @目录
   └──────────────────┘
```

是 `@` mention 的鼠标操作快捷入口，语义完全相同。

## Skills 在右侧面板的展示

Skills 显示在右栏底部，**只读**，用户不能在 UI 上操作：

```
[Skills]
  rust         (已打开)
  git
  api-style
```

- `(已打开)` 表示 AI 已 `open_file` 读取该 skill
- 是当前 session 的状态投影，操作权在 AI，用户通过聊天指示 AI
