# 监控文件区：目录树视图

> 从 [ui.md](ui.md) 延伸：将右栏监控文件区从扁平列表改为可折叠的虚拟目录树。

## 问题

当前监控文件区是扁平列表，文件少时清晰，但一旦打开 8-15 个文件（尤其分散在多个目录），列表变成一堆长路径，用户很难快速定位和分组理解。

## 方案：虚拟目录树

从 `open_files` 的路径集合中**动态构建**一棵最小目录树——只包含被打开的文件及其最短公共祖先，不展示未打开的文件。这不是工作区文件浏览器，而是"当前 AI 上下文中有哪些文件"的结构化视图。

---

## 视觉结构

```
[监控文件]  8 files · 5.9k tok
  ▾ src/
    ▾ auth/
        mod.rs          1.2k tok
        middleware.rs   0.8k tok
        session.rs      0.6k tok
    ▸ api/                              1.4k tok
      lib.rs            0.5k tok
  🔒 AGENTS.md          0.5k tok
  ▾ config/
    🔒 prod.toml        0.3k tok
      dev.toml          0.6k tok
```

- 折叠态的目录（`▸ api/`）右侧显示该目录下所有文件的 token 汇总
- 展开态的目录（`▾ src/`）不显示汇总，各文件独立显示
- 区域标题行显示文件总数 + token 总量

---

## 核心规则

### 1. 树只包含被打开的文件

不扫描磁盘，不展示未 open 的文件。树节点来源是 `open_files` 表中的路径，前端按路径分隔符拆分后构建嵌套结构。

### 2. 路径压缩

如果路径中存在只有单个子目录的中间层（如 `src/services/auth/`，其中 `services/` 只包含 `auth/`），压缩为 `src/services/auth/` 一行，减少无意义的层级嵌套。

### 3. 文件夹可折叠

- 点击文件夹名或箭头图标切换展开/折叠
- 折叠时显示目录 token 汇总，展开时隐藏汇总
- 折叠状态纯前端，存 `localStorage`，不影响后端数据

### 4. 锁定状态

- 文件级锁定只保留在行尾操作位展示：未锁定显示开锁，已锁定显示闭锁
- 锁定是文件级的，对应 `open_files` 表的 `locked` 字段
- 不在文件名前重复放置锁前缀，避免与行尾锁按钮形成双重状态提示
- 目录级不引入锁定语义；若目录下所有文件均 locked，可在目录名前用低对比度锁图标作视觉提示

### 5. 文件状态视觉

| 状态 | 视觉 |
|------|------|
| 正常 | 默认文字色 |
| locked | 行尾闭锁图标 + 略低饱和度文字 |
| `Deleted` | ~~删除线~~ + `text-dim` 色 |
| `Moved` | 旧路径删除线，新路径正常显示 |

### 6. 排序

同级内：目录在前，文件在后；同类按字母排序。

此排序是用户友好的结构化视图，与 AI 上下文中 `open_files` 的 `position` 顺序无关。

### 7. 工作目录相对路径

所有路径相对于 `workspace_root` 显示。工作目录外的文件显示绝对路径，归入一个特殊的「外部文件」分组，置于树底部。

## 右键菜单

| 目标 | 菜单项 |
|------|--------|
| 文件 | 关闭文件、锁定/解锁、在编辑器中打开 |
| 目录 | 关闭目录下所有文件、折叠/展开 |

"关闭目录下所有文件"是批量 `close_file`，locked 文件跳过并提示。

---

## 不做的事

- **不做完整文件浏览器**：不展示未打开的文件，浏览并打开是 `+ 打开文件` 按钮的职责
- **不做目录级锁定**：锁是文件粒度，与 `open_files.locked` 一一对应
- **不做拖拽排序**：树视图下拖拽语义复杂，MVP 不做
- **不改后端数据结构**：`open_files` 表不变，树是前端从 path 列表动态构建的纯视图层

---

## 实现要点

- 前端新增 `FileTree` 组件，输入 `Array<{path, tokens, locked, status}>`，输出可交互目录树
- `+ 打开文件` 按钮保留在区域底部，行为不变
- 可选提供「树视图 / 列表视图」切换，列表视图回退到原有扁平排序
