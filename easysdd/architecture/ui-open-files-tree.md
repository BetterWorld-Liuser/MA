# 监控文件区：材料清单视图

> 从 [ui.md](ui.md) 延伸：将右栏监控文件区组织成“当前工作台上摊开的材料清单”，而不是 IDE 式目录树。

## 问题

如果监控文件区按路径树或长路径列表展示，用户很难快速回答三件真正重要的事：

1. 现在摊开了哪些材料
2. 哪些材料属于同一组
3. 哪些材料成本高、状态特殊

这块区域不该主要回答“完整文件系统长什么样”。

## 方案：来源分组 + 材料组 + 文件项

从 `open_files` 的路径集合中**动态构建**一份分组清单：

- 先按来源分成 `Workspace` / `Outside workspace`
- 再在每个来源下提炼有意义的材料组
- 文件本身始终作为真正的主列表项

它仍然只展示当前已打开文件，但心智模型从“文件系统”切换成“当前工作台”。

## 视觉结构

```
9 files   20k   2 locked

Workspace
  AGENTS.md                         2.2k  🔒
  Skills                            5.1k
    content-strategy                3.2k
      SKILL.md                      3.2k
    copywriting                     1.9k
      SKILL.md                      1.9k

Outside workspace
  scratch.md                        0.4k
```

- 顶部摘要行只保留文件数、token 总量、锁定数
- 来源分组负责回答“材料从哪来”，不是重复路径细节
- 材料组像章节标题，不和文件名争注意力
- token 统一右对齐，作为尾注看；锁定图标只表达状态

---

## 核心规则

### 1. 清单只包含被打开的文件

不扫描磁盘，不展示未 open 的文件。所有分组都从 `open_files` 路径集合动态推导。

### 2. 先按来源分组

- 工作目录内文件进入 `Workspace`
- 工作目录外文件进入 `Outside workspace`
- 外部文件默认不显示绝对路径；完整路径只在 tooltip / hover 中出现

### 3. 材料组只在有意义时出现

- 单文件直接显示为文件项，不额外加“目录壳”
- 只有当多个文件共享一段有意义的目录语义时，才提升为材料组标题
- 单子目录长链可压缩成一个组名（如 `src/services/auth`），减少无效层级

### 4. 只有多文件组可折叠

- 折叠只服务“这一组内容很多，一眼扫不完”的场景
- 单文件组不提供折叠控制，避免用户先和结构搏斗
- 展开状态纯前端，存 `localStorage`，不影响后端数据

### 5. 锁定状态

- 锁定是文件级的，对应 `open_files` 表的 `locked` 字段
- 锁只作为轻量状态图标，不再占据主操作位
- 被锁定文件不显示关闭操作

### 6. 文件状态视觉

| 状态 | 视觉 |
|------|------|
| 正常 | 文件名作为主视觉 |
| locked | 行尾闭锁图标 + 略低饱和度文字 |
| `Deleted` | 文件名删除线 + 轻状态文案 |
| `Moved` | 文件名删除线 + 行内显示新文件名 |

### 7. 排序

同级内：材料组在前，文件在后；同类按字母排序。

此排序服务“扫读当前材料”，与 AI 上下文中 `open_files` 的 `position` 顺序无关。

### 8. 工作目录相对路径

工作目录内文件相对于 `workspace_root` 推导组结构。工作目录外文件归入 `Outside workspace`，但默认隐藏绝对路径。

---

## 交互

当前版本保持克制：

- 常态只展示文件名、轻量状态和 token
- 完整路径通过 tooltip 提供
- 关闭操作只在 hover 时出现
- 若组确实包含多份材料，才允许折叠 / 展开

如后续补充右键菜单，也应继续遵循“管理动作按需出现，不干扰常态扫读”。

---

## 不做的事

- **不做完整文件浏览器**：不展示未打开的文件，浏览并打开仍是 `+ 打开文件` 的职责
- **不默认显示绝对路径**：路径细节退到 tooltip，避免和文件名抢注意力
- **不做目录级锁定**：锁是文件粒度，与 `open_files.locked` 一一对应
- **不改后端数据结构**：`open_files` 表不变，材料清单是前端从 path 列表动态构建的纯视图层

---

## 实现要点

- 前端采用“来源分组 + 材料组 + 文件项”的递归清单组件
- 多文件组的折叠状态由前端持久化
- `+ 打开文件` 按钮保留在区域底部，行为不变
