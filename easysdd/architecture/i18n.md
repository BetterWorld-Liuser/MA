# 国际化（i18n）

## 设计目标

让 March 的 UI 文本支持多语言切换，初期覆盖中文（zh-CN）和英文（en）。方案要求零外部依赖、代码量极小、接入侵入性低。

---

## 为什么不用 vue-i18n

March 当前所有 UI 文本都是简单字符串或带 1-2 个变量的模板，不涉及复数规则、日期/数字格式化、ICU MessageFormat 等高级特性。引入 vue-i18n（~40KB gzipped ~14KB）会带来大量用不到的 API 表面积。项目原则是"不为假设性需求引入抽象"，因此选择自管理方案。如果将来文本复杂度增长到需要 ICU MessageFormat，再迁移。

---

## 核心设计

### 全局 composable

```
src/i18n/
  index.ts          # useI18n composable + 全局状态
  locales/
    zh-CN.ts        # 默认语言，导出嵌套对象
    en.ts           # 英文
```

`useI18n()` 返回：

| 名称 | 类型 | 说明 |
|------|------|------|
| `t` | `(key: string, params?: Record<string, string \| number>) => string` | 按 key 查当前语言字符串，支持 `{name}` 占位符替换 |
| `locale` | `Ref<Locale>` | 当前语言，全局 reactive ref，所有组件共享 |
| `setLocale` | `(locale: Locale) => void` | 切换语言，同时写 localStorage |

其中 `Locale = 'zh-CN' | 'en'`。

### 运行机制

- `locale` 是模块级 `ref`，由 `useI18n()` 返回同一个引用，不依赖 provide/inject
- 初始化时从 `localStorage` 读取，缺省 fallback 到 `zh-CN`
- `setLocale` 同步更新 ref 和 localStorage
- `t(key)` 按 `.` 分隔 key，沿嵌套对象逐层查找；找不到时返回 key 本身（开发阶段可 console.warn）
- 占位符替换：遍历 params，将字符串中的 `{paramName}` 替换为对应值

## 组件接入方式

### 模板中使用

```vue
<script setup lang="ts">
import { useI18n } from '@/i18n'
const { t } = useI18n()
</script>

<template>
  <Button>{{ t('common.cancel') }}</Button>
  <p>{{ t('taskList.deleteTask', { name: task.name }) }}</p>
</template>
```

### script 中的静态选项

对于 `sectionOptions`、`themeOptions` 这类在 `<script setup>` 中定义的数组，需要改为 computed 以响应语言切换：

```ts
const sectionOptions = computed(() => [
  { value: 'appearance', label: t('settings.appearance.label'), ... },
  ...
])
```

### 不需要 i18n 的文本

以下文本不纳入翻译体系：

- **AI 面向的文本**：system prompt、tool description 等，这些是给模型看的，不是给用户看的
- **技术标识符**：provider type label（`"OpenAI"`, `"Anthropic"`）、模型名称、文件路径
- **Debug 面板的 tab 名**（`Overview`, `Context`, `Request`, `Response`, `Tools`）——这些是开发者工具的技术术语，保持英文

## 语言切换 UI

在 Settings → 外观分区中，在"主题"卡片下方新增"语言"卡片，样式与主题卡片一致：

- 两个选项：`中文` / `English`
- 点击即时切换，持久化到 localStorage
- key: `ma-locale`（与 theme 的 `ma-theme` 对齐）

## 迁移策略

分批替换，每个组件单独 commit：

1. 建立 `src/i18n/` 目录，实现 composable + 两个语言文件
2. Settings → 外观分区加语言切换 UI
3. SettingsPage 全量替换（文本最密集）
4. ChatPane + useChatComposer
5. NoteEditorDialog / ConfirmActionDialog
6. TaskList
7. ContextPanel + 四个 Context 子组件
8. App.vue 顶层文本

每完成一个组件，确保两种语言都能正常渲染。
