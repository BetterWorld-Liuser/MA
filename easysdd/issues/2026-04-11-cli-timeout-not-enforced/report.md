# AI 命令超时与手动中断均不生效 Issue Report

> 阶段: 阶段一(问题报告)
> 状态: 草稿
> 创建日期: 2026-04-11
> 严重程度: P1

## 1. 问题现象

在 March 的 AI 命令行工具中，当技能自动触发命令 `npx skills find marketing` 时，即使工具侧已经设置了 `timeout 20s`，命令也不会按预期超时结束：

- 该现象稳定复现
- 命令运行超过 20 秒后不会自动终止
- 实际会持续到大约 60 秒仍未结束
- 用户尝试手动中断也无法停止执行
- UI 没有显示超时、错误或过程输出，只是表现为一直卡住

当前问题仅描述外部可见行为，不包含对根因的判断。

## 2. 复现步骤

1. 打开 March 的 AI 命令行工具
2. 在对话中触发技能自动执行命令 `npx skills find marketing`
3. 观察工具显示的命令参数中包含 `timeout 20s`
4. 等待命令执行结果，并在约 60 秒时尝试手动中断
5. 观察执行状态与 UI 表现

观察到: 命令超过 20 秒后没有超时返回，持续运行到约 60 秒仍未结束；UI 无输出且卡住，手动中断也无效

复现频率: 稳定复现

## 3. 期望 vs 实际

**期望行为**: AI 发起 `npx skills find marketing` 且设置了 `timeout 20s` 后，命令应在约 20 秒达到超时时被终止，并返回明确的超时结果；如果用户点击中断，也应能及时停止执行。

**实际行为**: 命令超过 20 秒后没有被终止，持续运行到约 60 秒仍未结束；UI 没有显示超时、报错或进度输出，点击手动中断也无效，界面表现为一直卡住。

## 4. 环境信息

- 涉及模块/功能: AI 命令行工具，技能自动调用命令执行链路
- 相关文件/函数: 待定
- 运行环境: 正式包
- 其他上下文: Windows 桌面端；通过技能自动调用命令，不是用户手动执行 shell；近期没有改过相关代码，属于之前就存在的问题

## 5. 严重程度

**P1** — 命令执行的超时控制、取消能力和结果反馈属于 AI 工具的核心交互链路。当前问题会导致命令长时间悬挂、无法及时中断，并让界面卡住，属于严重影响核心使用体验的问题。
