# MEMORY 面板未保留本轮已激活记忆 Issue Report

> 阶段:阶段一(问题报告)
> 状态:已确认
> 创建日期:2026-04-11
> 严重程度:P2

## 1. 问题现象

在 AI 回复生成过程中，用户能明显看到 memory 被调用，或界面里短暂出现命中的 memory；但当回复结束、界面稳定后，右侧 `MEMORY` 区域又显示为空，看不到这一轮回复里实际激活过哪些 memory。

## 2. 复现步骤

1. 刚打开一个 task，此时右侧 `MEMORY` 区域默认没有内容。
2. 发送一条会触发 memory 检索或调用的消息。
3. 在回复生成过程中，观察到界面里会短暂出现命中的 memory，或能明显看见 memory 被调用。
4. 等回复结束后，右侧 `MEMORY` 区域重新显示为空。
5. 观察到:这一轮回复实际激活过哪些 memory 没有在回复结束后继续保留展示。

复现频率:待补充（当前已确认可观察到该现象）

## 3. 期望 vs 实际

**期望行为**:回复结束后，右侧 `MEMORY` 区域仍然展示这一轮实际激活过的 memory 列表，让用户能够回看本轮命中了哪些记忆。

**实际行为**:回复过程中能看到 memory 被调用或短暂出现，但回复结束后列表消失，右侧看起来像这一轮没有激活过任何 memory。

## 4. 环境信息

- 涉及模块/功能:右侧 Context 面板的 `MEMORY` 展示
- 相关文件/函数:待定
- 运行环境:不确定

## 5. 严重程度

**P2** — 不直接阻塞对话主流程，但会影响 memory 能力的可观测性，用户无法在回复完成后确认这一轮到底激活了哪些记忆，削弱对系统行为的理解与信任。
