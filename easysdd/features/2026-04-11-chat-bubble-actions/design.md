## 0. 需求摘要

- **做什么**：在聊天区每个消息气泡下方增加一块固定占位的快速操作区，首个按钮为复制消息内容的 icon 按钮。
- **为谁**：在聊天区阅读和复用 AI 回复的用户。
- **成功标准**：鼠标移入消息气泡下方的操作区时显示复制按钮，移出时按钮隐藏但操作区仍占位，消息布局不闪烁；点击复制按钮可复制该条消息内容。
- **明确不做**：本次不引入除复制之外的新消息操作；不处理富文本到 markdown 以外的特殊导出格式；不改变现有消息折叠/引用/中断交互模型。

## 1. 设计方案

### 改动点

| 文件 | 改动内容 |
|---|---|
| `src/components/chat/TurnGroup.vue` | 在 assistant 气泡下方新增统一的消息操作区容器；基于 turn 文本内容控制复制按钮显隐、可用态与点击复制逻辑。 |
| `src/components/chat/UserMessageBubble.vue` | 在 user 气泡下方接入同一套消息操作区，保证不同气泡交互一致。 |
| `src/styles/main.css` | 扩展 `message-actions` / `message-copy-button` 相关样式，形成“始终占位、进入预留区时显现”的操作区，并预留后续更多 icon 按钮的布局空间。 |
| `src/components/ContextPanel.vue` | 复用现有 `navigator.clipboard.writeText(...)` 调用方式作为聊天复制交互的参考，保持桌面端复制能力一致。 |

### 新增类型（如有）

本次不新增共享类型；继续使用 `Turn`、`AssistantTimelineEntry` 现有结构，在组件内部派生可复制文本。

### 边界情况

- **消息仍在 streaming**：操作区仍占位，但复制按钮禁用，避免复制到半成品内容。
- **无文本内容**：操作区仍占位，但复制按钮禁用。
- **hover 离开**：只切换按钮内容透明度/指针事件，不移除 DOM 和高度，避免气泡闪烁或跳动。
- **键盘可达性**：除 hover 外，也允许在操作区或按钮获得 focus 时保持可见，避免纯鼠标交互限制。

## 2. 验收标准

### 功能验收

- [ ] 打开一条已完成的 AI 消息和一条用户消息，鼠标移入各自气泡下方预留区后，出现复制 icon 按钮；移出后按钮隐藏，但气泡与下方间距不发生跳动。
- [ ] 点击复制按钮后，系统剪贴板写入对应消息内容；再次粘贴可得到与消息文本一致的内容。
- [ ] 同一操作区在视觉上保留可扩展位置，而不是复制按钮直接贴在气泡外缘，后续可继续横向加按钮。

### 异常与边界

- [ ] AI 消息处于 `streaming` 状态时，操作区仍保留占位，但复制按钮不可点击。
- [ ] 某条消息没有可复制文本时，复制按钮保持禁用，不报错、不抖动。

### 回归检查

- [ ] 用户消息气泡样式与布局保持一致，仅新增底部操作区。
- [ ] 现有 AI 气泡正文、折叠/展开、“引用”和“中断 turn”按钮行为不受影响。

## 3. 推进步骤

1. **梳理消息动作区结构**：在 `TurnGroup.vue` 与 `UserMessageBubble.vue` 中抽出可复制文本与操作区显示条件 → 退出信号：两类气泡模板里都出现固定占位的 actions 区域。
2. **实现复制交互**：接入剪贴板写入和按钮禁用态 → 退出信号：两类消息都具备可执行的 `copy` 行为，streaming/无文本时自动禁用。
3. **补齐 hover 占位样式**：在 `main.css` 中实现“常驻高度、进入预留区时显示内容、默认隐藏内容”的样式策略 → 退出信号：视觉上无闪烁，可扩展多个 icon 按钮。
